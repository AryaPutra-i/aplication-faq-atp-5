FAQ ACCORDION

1. Apa perbedaan dasar antara UI dan UX Design ?

UX (User Experience) berfokus pada rasa dan fungsi (apakah aplikasi mudah digunakan dan solutif), sedangkan UI (User Interface) berfokus pada tampilan visual (warna, tipografi, dan tata letak) agar produk terlihat menarik.

2. Apakah seorang UI/UX Designer wajib bisa coding?

Tidak wajib. Fokus utamanya adalah riset pengguna, merancang alur (wireframing), dan membuat purwarupa (prototyping). Namun, memahami dasar teknis akan menjadi nilai tambah besar saat berkomunikasi dengan developer.

3. Apa perbedaan SysAdmin dengan DevOps ?

SysAdmin secara tradisional fokus pada pemeliharaan, konfigurasi, dan troubleshooting infrastruktur server. Sedangkan DevOps fokus pada otomasi alur kerja pengembangan (pipeline CI/CD) untuk menjembatani tim pengembang (developer) dengan operasional agar perilisan aplikasi lebih cepat dan sering.

4. Apa perbedaan mengelola server On-Premise dan Cloud ?

Pada On-Premise, SysAdmin bertanggung jawab penuh mulai dari fisik server (hardware, kabel, listrik, pendingin) hingga perangkat lunak di lokasi kantor sendiri. Pada Cloud (seperti AWS/Azure), infrastruktur fisik diurus oleh penyedia layanan, sehingga SysAdmin hanya fokus mengelola konfigurasi virtual dan sistem operasi.

5. Bagaimana urutan belajar (roadmap) yang ideal untuk SysAdmin pemula?

Fokuslah pada 3 tahap berurutan: Pertama (Fondasi) Kuasai Terminal Linux dan Jaringan Dasar (IP/DNS). Kedua, (Layanan) Belajar instalasi Web Server dan Database. Ketiga, (Otomasi) Pelajari dasar Scripting (Bash) untuk efisiensi.

6. Apa tugas utama seorang Technical Writer ?

Tugas utama Technical Writer adalah membuat dokumentasi teknis yang jelas dan mudah dipahami, seperti panduan pengguna, manual instalasi, dan dokumentasi API. Mereka menjembatani kesenjangan antara informasi teknis kompleks dan kebutuhan pengguna akhir dengan menyajikan informasi secara terstruktur dan komunikatif.

7. Kapan waktu terbaik mulai menulis dokumentasi ?

Waktu terbaik untuk mulai menulis dokumentasi adalah sejak awal pengembangan produk. Dengan mendokumentasikan fitur dan proses sejak tahap awal, informasi akan lebih akurat dan lengkap. Selain itu, dokumentasi yang dibuat secara bertahap akan lebih mudah diperbarui dan disesuaikan dengan perubahan produk.
